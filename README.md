Digitalbank Landing Page – Bootstrap Implementation
Project Overview
This project is a responsive landing page built as part of a challenge from Frontend Mentor. The goal of the challenge was to recreate a provided design using modern frontend development practices while ensuring responsiveness across multiple screen sizes.
The project was developed using HTML, CSS, and Bootstrap. The focus of the exercise was to practice working with Bootstrap’s grid system, layout utilities, and responsive design principles.

The landing page includes:
A responsive navigation bar with a mobile modal menu
A hero section with an image overlay
Feature sections highlighting product benefits
A responsive article grid
A footer with navigation and social icons

Approach and Customizations
The project was built using Bootstrap’s grid system and flex utilities to structure rows and columns across different screen sizes. Several design elements required customization beyond the default framework behavior.
Customizations included:
Creating full-width sections using container-fluid
Implementing responsive column stacking and ordering
Overlaying images in the hero section using Bootstrap cards and card-img-overlay
Building a mobile navigation modal triggered by the hamburger menu
Using responsive utility classes to adjust alignment and layout across breakpoints
The goal was to stay as close as possible to the original design while learning how Bootstrap handles layout constraints and responsive behavior.

Challenges and Problem Solving
As an introductory project to Bootstrap, the challenges were numerous. Many hours were spent learning how to make rows span the full width of the screen rather than being constrained to a container. Other challenges included correctly overlaying images in the hero section and ensuring that the layout responded properly across screen sizes.
My approach to solving these problems typically began with research. I would first search Google or ChatGPT with questions such as “How do I…”, “Why didn’t this work?”, or “What happens if I…”. Quite honestly, chat felt like a really good coding patrner on this project.  My understanding of Bootstrap is much stronger as a result of the exchanges of ideas and experimentation that continuously happened.  I also relied heavily on reading the Bootstrap documentation to better understand how the framework’s classes and layout system function.
If documentation and written explanations were not enough, I often turned to YouTube tutorials to gain additional insight.  In particular https://youtu.be/Wqu-d_b3K-0. Bootstrap Grid System Tutorial | Bootstrap 5 was helpful in getting me started. My learning process tends to be nonlinear, moving back and forth between documentation, experimentation, and external resources until the concept becomes clear.  

Future Improvements
This submission was completed under time constraints, and there are several design refinements that could still be improved. Given more time, I would continue refining spacing, margins, and padding to better match the original design.
Some of the issues I encountered and that still remain was my inability to achieve the correct left margin, the gap between my Latest Article  heading and the content are off and my Why Choose My Digitalbanking heading in the first row does not center in mobile view which it should. Addressing these types of visual refinements sometimes required creative solutions to work within Bootstrap’s built-in structure and limitations. With more time, I would continue refining these details and explore additional layout techniques to improve overall visual precision.
